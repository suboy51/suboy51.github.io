Hexo 架站筆記：常用指令與基礎操作

這是我建立個人網站的第一篇文章。為了怕以後金魚腦忘記怎麼操作，特地把常用的 Hexo 指令與流程記錄下來。

1. 寫作標準流程 (Workflow)

經營部落格最常用的四個步驟：

  1. 新增文章：建立 Markdown 檔案。
  2. 撰寫內容：使用 VS Code 編輯。
  3. 本地預覽：在電腦上檢查版面。
  4. 發布上線：推送到 GitHub Pages。

2. 常用指令大全

這裡整理了最常用的縮寫指令，不用每次都打完整單字。

功能 完整指令 縮寫 (推薦) 備註
新增文章 hexo new "標題" hexo n "標題" 檔案會產生在 source/_posts
啟動伺服器 hexo server hexo s 預覽網址通常是 localhost:4000
清除快取 hexo clean hexo cl 救命神技！ 版面跑掉時先按這個
生成檔案 hexo generate hexo g 將 Markdown 轉為 HTML
部署發布 hexo deploy hexo d 上傳到 GitHub

🔥 懶人組合技

我最常用的發布指令（生成 + 部署）：

hexo g -d
